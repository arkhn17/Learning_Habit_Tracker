# Learning Habit Tracker

職業訓練校の企業実習/自主制作課題として開発中のWebアプリです。  
日々の学習内容や気づきを記録し、自己成長を可視化することを目的としています。

---

## 🗓 2日目（UI構想・画面遷移図作成）
（Day2：環境構築とUI構想の作成中）  
- VSCodeでの作業環境を整備  
- `drawio/ui構想.drawio` を作成し、UI構想を設計中  
- GitHub初回コミット（README.md、企画書骨子.mdなど）

### ✅ 実施内容
- Draw.io（diagrams.net）の環境確認  
  - 図形の作成、テキスト入力、色変更、保存操作などを確認  
- UI構想（画面遷移図）の作成  
  - 「トップページ」と「新規学習記録ページ」を大枠で設計  
  - トップページ内の要素を整理：  
    - 表示：[学習記録一覧（テーブル）] → 日付／内容／気付き／進捗  
    - セクション：[目標・達成状況]  
    - ボタン：[新規学習記録]  
  - 新規学習記録ページ内の要素：  
    - 入力フォーム：[日付／学習内容／気付き／目標進捗]  
    - ボタン：[登録]（トップページに戻る）  
    - ボタン：[トップページ]（トップページに戻る）  
- ページ構造を理解し、drawio上でUI全体をラフ設計  

### 学び・気付き・躓きポイント
- 画面遷移図の描き方（ページ＝大きな長方形、要素＝内側の図形）を理解。  
- 「ボタン」「入力フォーム」などの命名ルールを統一する重要性を理解。  
- 「目標・達成状況」を独立したセクションとして整理したことで構造が明確になった。  
- ログイン機能が不要な理由（ローカル保存の想定）も理解。  
- 線（遷移矢印）は未完だが、ページ構造は完成。
